# Первый запуск — проверка окружения

Это первое, что ты делаешь, когда владелец только открыл эту папку. Цель — убедиться,
что всё, что нужно среде, на месте, и по-человечески подсказать, если чего-то не хватает.
Потом переходишь к знакомству (`prompts/interview.md`).

Сначала определи систему владельца (`macOS` или `Windows`) — от неё зависят подсказки.

## Что проверить

1. **Для самой среды (второй мозг) ничего лишнего не нужно.** Ты читаешь и пишешь файлы
   в этой папке — этого достаточно. Если владелец пришёл собрать второй мозг — сразу
   переходи к интервью, не гоняй его по установкам.

2. **Node нужен только для продвинутого** — если владелец хочет запускать агентов по
   расписанию, ставить коннекторы или работать в Codex. Тогда проверь: `node --version`.
   - Версия есть → всё готово, ничего не делай.
   - Команды нет → объясни простыми словами и дай путь под его систему:
     - **macOS / Windows одинаково:** зайди на `nodejs.org`, скачай версию **LTS**, открой
       скачанный файл (`.pkg` на маке, `.msi` на винде) и пройди установку кнопкой «Далее».
     - Дождись, проверь `node --version` ещё раз.

3. **Python тоже нужен только для продвинутого** — многие скрипты и рендер (например,
   картинки, обработка данных, автоматизации Дня 2) работают на Python. Проверь:
   `python3 --version`.
   - Версия есть (3.10+) → всё готово, ничего не делай.
   - Команды нет → дай путь без прав администратора:
     - **macOS / Windows одинаково:** зайди на `python.org/downloads`, скачай последнюю
       версию, открой файл (`.pkg` на маке, `.exe` на винде — на винде отметь галочку
       «Add Python to PATH») и пройди установку.
     - Дождись, проверь `python3 --version` ещё раз.

4. **Пакетные менеджеры (Homebrew на маке и т.п.) не ставим.** Их установка требует прав
   администратора (`sudo`), а по правилам (`AGENTS.md`) команды с `sudo` мы не запускаем.
   Node и Python ставятся своими установщиками с сайта без этого — этого достаточно.

## Границы

- `sudo` и права администратора не используешь. Если что-то по-честному их требует — не
  делаешь сам, а даёшь владельцу точный путь: что скачать и куда нажать.
- Не объявляй «всё установлено», не проверив командой. Нет данных — проверь или спроси.
- Windows-путь не тестировался вживую: если команда или путь не сходятся с тем, что видит
  владелец, не выдумывай — сверься с `code.claude.com/docs` и спроси владельца, что у него
  на экране.

Когда окружение в порядке — переходи к `prompts/interview.md`. После знакомства можешь
предложить `prompts/meta-skills.md` (голосовой ввод + скриншот в буфер) — по желанию.
