# WhyNot Starter — твоя AI-среда за 15 минут

Это не курс и не программа. Это папка, которая превращает Claude Code в твоего
личного ассистента: он знает, кто ты, чем занимаешься, и становится умнее с каждым
разговором.

## Как запустить (3 шага)

1. **Открой эту папку в VS Code**: File → Open Folder → выбери её.
2. **Открой своего агента** (Claude Code, Codex — что у тебя стоит) и напиши:
   `прочитай AGENTS.md и поздоровайся`.
3. **Скопируй в чат содержимое `prompts/interview.md`** — агент по одному вопросу
   наполнит папки `you/` и `business/` твоими фактами. Займёт 15–30 минут, отвечать
   можно голосом или коротко, «не знаю» — нормальный ответ.

## Работает с любым агентом

Инструкция среды лежит в `AGENTS.md` — это общий стандарт, который агенты читают сами:

- **Claude Code** — читает `CLAUDE.md`, тот ведёт в `AGENTS.md`. Плюс автоматически
  действуют запреты из `.claude/settings.json`.
- **Codex (OpenAI)** — читает `AGENTS.md` напрямую, ничего настраивать не надо.
  ⚠️ Запреты из `.claude/settings.json` на Codex НЕ действуют — держи режим
  подтверждений (Ask/Auto), не включай Full Access, пока учишься.
- **Qwen Code / Gemini CLI и другие** — если агент сам не подхватил, первой командой
  напиши: «прочитай AGENTS.md и работай по нему».

Всё. Дальше просто работай: проси написать текст, разобрать идею, собрать штуку.
Агент сам берёт контекст из папок.

## Что тут лежит

| Папка/файл | Зачем |
|------------|-------|
| `AGENTS.md` | Инструкция агенту — любой агент читает её первой |
| `CLAUDE.md` | Указатель на AGENTS.md для Claude Code |
| `you/` | Кто ты: профиль, голос, уроки, которые агент выучил о тебе |
| `business/` | Твоё дело: продукт, аудитория, цели |
| `prompts/` | Готовые промты: интервью, самопроверка, проверка чужих скиллов |
| `handoff.md` | «Где остановились» — агент пишет сюда в конце сессии, читает в начале |
| `.claude/settings.json` | Защита: агент не может читать твои пароли и удалять всё подряд |

## Правила безопасности

1. **Пароли и ключи — только в файле `.env`.** Агент настроен их не читать, git — их не хранить.
2. **Режим работы — обычный (Default)**, не «Bypass permissions» — это твоя главная защита:
   агент спрашивает разрешение на каждую команду, и ты видишь, что он делает. Запреты в
   `.claude/settings.json` — второй пояс, а не замена этому режиму.
3. **Чужой скилл из интернета — сначала проверка.** Промт в `prompts/skill-security-check.md`.
4. **Появились платные API-ключи — усиль защиту**: промт в `prompts/env-guard.md` ставит
   блокирующий hook, который перехватывает чтение секретов даже из чужих скиллов.

Честная оговорка: эта папка — ремни безопасности, не сейф. Не включай Bypass на компьютере,
где лежат рабочие пароли и доступы, и не храни в папке проекта ничего секретного сверх `.env`.

## Частые вопросы

**Это привязано к какому-то сервису?** Нет. Обычная папка с текстовыми файлами —
твоя, локальная, переносимая.

**Что если агент ошибается в фактах обо мне?** Поправь его словами. По правилам из
`CLAUDE.md` он запишет урок в `you/feedback.md` и больше так не сделает.

**Можно вести несколько проектов?** Да: заведи внутри подпапки `project-1/`, `project-2/` —
`you/` и `business/` общие на всех.
